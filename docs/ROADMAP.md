# TiaLynn 路线图

> 6 个里程碑，每个里程碑都是可独立 demo 的状态。  
> 当前阶段：**M0 — 地基重置**（v0.4.x）

---

## M0 — 地基重置（v0.4.x）

**目标**：把现有混乱代码砍掉重组，建立干净的模块边界。

### 任务
1. 阅读现有代码，识别可保留的核心逻辑 ✅（docs/M0_INVENTORY.md）
2. 按五大域搭新骨架
3. Live2D 模型路径完全配置化（支持任意来源目录）
4. 事件总线（前后端）
5. 砍 8 项过度设计代码
6. 灵魂 YAML 拆为 4 份
7. 验证最小对话链路

### 完成标准
- 项目能跑起来，Live2D 显示，能切换模型
- 模块边界清晰，跨模块只通过事件通信
- 跑通最小对话链路（哪怕只是 echo）
- 文档完整

## M1 — 能聊（v0.5.x）

**目标**：能用文字流畅对话，表情会变。

### 任务
- `brain/providers/anthropic.rs` —— Claude API 主路径（tool_use 支持，给 M4 预备）
- `brain/providers/ollama.rs` —— 本地兜底
- LLM Provider 统一接口
- 流式输出 + 气泡显示（已有）
- 三层 prompt 架构稳定（已有，搬域）
- 表情情绪映射 + JSON emotion 协议（已有）

### 完成标准
- 切换 LLM provider 在菜单完成
- 输入 → 流式输出 → 表情联动 < 1.5s 首 token

---

## M2 — 能听能说（v0.6.x）

**目标**：戴耳机能完整语音对话。

### 任务
- macOS say 验证稳定（已有）
- CosyVoice 2 sidecar 实装 + voice clone（你的 example_voice）
- whisper.cpp 替代 faster-whisper（更快，C++ 原生）
- 嘴型同步（已有 lipSync.ts，需精度提升）
- STT F8 / 鼠标按钮 push-to-talk（已有）

### 完成标准
- 按 F8 → 说话 → 她用她的声音回答 → 嘴型同步
- 端到端延迟 < 3s

---

## M3 — 能记（v0.7.x）

**目标**：她能记得你昨天说过的事。

### 任务
- ChromaDB 替代当前全表 cosine 召回
- 共同回忆导入：聊天记录 → embed → 入库
- 启动时主动召回相关记忆
- 凝练 tick 优化（已有 distill.rs，需要更智能的 fact 抽取）
- learned_traits.yaml 自动写入

### 完成标准
- 上次说过的事，下次启动会被提及
- 1 个月用量下召回延迟 < 200ms

---

## M4 — 能干活（v0.8.x）★ **项目灵魂**

**目标**：她能帮你打开文件、跑命令、查代码。

### 任务
- MCP client 集成（@modelcontextprotocol/sdk）
- 接入官方 MCP servers：filesystem / shell / browser / git
- 工具调用 UI：意图展示 + 批准/拒绝对话框
- 安全沙盒：默认敏感操作需确认（白名单可放行）
- 工具能力描述自动生成（feed 给 LLM）

### 完成标准
- 对话："帮我打开 a.txt" → 她说"好" → 弹批准 → 通过 → 文件打开
- "帮我看 README 里 voice clone 怎么用" → 她读文件 → 在气泡里讲解

---

## M5 — 能主动（v0.9.x）

**目标**：她真的像个"在那儿"的存在。

### 任务
- 时段感知（早晚不同语气）
- 空闲检测（鼠标不动多久）
- 屏幕感知（你在用什么应用，写代码？看视频？）
- 主动开口策略（频率可调、不烦）
- 基于观察的建议（"看代码 1 小时了，要不要休息？"）
- 桌面散步（克制版，每 5-15 min 走一次）

### 完成标准
- 不主动配合用 1 天，发现她确实"住在桌面上"

---

## M6 — 能进化（持续）

**目标**：用一个月后，她和第一天明显不同。

### 任务
- 习惯学习写入 learned_traits
- 记忆强化与遗忘（recall_count + last_recall 已有 schema）
- 自我反思 tick：定期问自己"我了解 master 什么新东西"
- 灵魂包导出/导入（迁移到新机器）
- 元认知：她能查询"我记得哪些事"

### 完成标准
- 一个月后导出"灵魂包"，能在新机器还原她

---

## 不在路线图的（明确不做）

| 功能 | 原因 |
|---|---|
| 像素级穿透 | 与窗口管理冲突，矩形穿透已够用 |
| 复杂 idle 状态机 | 过度设计，靠 LLM 驱动表情 + Live2D motion 即可 |
| 自定义 Live2D 动作绑定 UI | 价值低，等真正需要时再做 |
| 在线遥测 / 用户统计 | 隐私优先，永不做 |
| 多角色互动 | 偏离"她是唯一的"核心理念 |
